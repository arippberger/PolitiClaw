# Legacy Docs Audit

This page tracks the biggest categories where earlier planning material drifted away from the runtime that actually shipped.

## Claims Retired From The Published Docs

- dashboard and HTTP route surfaces
- long-running background service claims
- send-on-your-behalf outreach behavior
- broad provider coverage that only exists as declared config placeholders

## Claims Migrated Into Current Docs

- local-first storage
- plugin-managed monitoring
- federal bill and House vote coverage
- ballot preparation and election guidance
- draft-only outreach

## Claims Still Pending Before They Can Become Public Docs

- additional provider integrations that are still declared-only in the schema
- a live production transport for guarded narrative web search
- any new runtime surface beyond the current tool, storage, source, skill, and cron layers

Treat the generated reference pages as the gate for when a pending capability is ready to be described as live.
